# Read→Do UI/UX 规格
Version: 0.4
Last Updated: 2026-02-17

---

## 1. 设计原则

| 原则 | 说明 |
|------|------|
| **速度优先** | Capture 3 秒完成，Inbox 一屏决策 |
| **可解释** | 每条建议都有理由，用户知道"为什么" |
| **可编辑** | AI 产物不是只读的，用户随时可改 |
| **关闭即安心** | Capture 后即可放心关闭 Tab |
| **批量高效** | 多选 + 一键操作，快速清理队列 |

## 3. Chrome Extension Popup

### 3.1 布局

紧凑弹窗，宽 360px，高度自适应。

```
┌─────────────────────────────────────┐
│   ◆ Read→Do              ✕         │
│   ┌─────────────────────────────┐   │
│   │  📄 Article Title Here      │   │
│   │     blog.example.com        │   │
│   └─────────────────────────────┘   │
│   Why save this?                    │
│   ┌─────────────────────────────┐   │
│   │  e.g. "想了解他们的架构设计"  │   │
│   └─────────────────────────────┘   │
│   ┌─────────────────────────────┐   │
│   │         💾  Save            │   │
│   └─────────────────────────────┘   │
└─────────────────────────────────────┘
```

**成功状态：** ✓ Captured! · Safe to close this tab. · Open Inbox →

### 3.2 交互流程

1. 点击插件图标 → 弹出 Popup
2. 自动填充 title + domain（只读）
3. 光标聚焦 intent 输入框（placeholder 引导）
4. 输入 intent → 点 Save（intent 可选但鼓励）
5. 成功：切换为成功画面
6. 可点击 "Open Inbox →" 跳转 Web App

### 3.3 状态处理

- **保存中**：按钮 spinner + "Saving..."，禁用交互
- **成功**：绿色对勾 + 引导文案
- **失败**：底部红色 toast + Retry 按钮

│          │                                  │
│  ◆ Read  │   Inbox                          │

│          │   │  Item Card               │   │
│  📦 Arch │   └──────────────────────────┘   │
│          │   ┌──────────────────────────┐   │
│          │   │  Item Card               │   │

**Sidebar 导航：**
- **Inbox**（默认，带未处理数量 badge）

### 4.2 Inbox 列表页

#### 工具栏

```
┌──────────────────────────────────────────────┐
│  [🔍 搜索标题、域名、意图...]   [☐ 选择模式]  │
└──────────────────────────────────────────────┘
```

- **搜索**：输入即搜（300ms 防抖），匹配 title / domain / intent_text
- **选择模式**：点击切换，卡片左侧出现 checkbox

#### 列表分组

READY 内容按 priority 分组，组内按 score 降序。PROCESSING / FAILED 在顶部。

```
⏳ Processing (2)     ← 骨架屏加载态
🟢 Do first           ← 优先行动
🔵 Plan it
⚪ Skim it
🔴 Let go
```

#### Item 卡片

```
┌──────────────────────────────────────────────────┐
│  [☐]  Do first                            85     │
│       How We Redesigned Our API Gateway          │
│       engineering.example.com · web              │
│       "想了解他们网关的限流和熔断方案"                │
│       ✦ 与你关注的系统设计方向高度匹配                │
│       ✦ 包含可直接复用的架构决策模式                  │
│                                     2 hours ago  │
└──────────────────────────────────────────────────┘
```

- 选择模式下左侧显示 checkbox，选中后卡片高亮
- 悬停：卡片上浮 + 阴影加深
- 点击（非选择模式）：进入详情页

#### 批量操作栏

选择模式下有选中项时，底部浮现操作栏：

```
┌──────────────────────────────────────────────────┐
│  已选 3 项      [ 📦 归档选中 ]  [ 🗑 删除选中 ]  │
└──────────────────────────────────────────────────┘
```

- 归档：一键将选中项状态改为 ARCHIVED
- 删除：确认对话框后级联删除

#### 特殊状态

| 状态 | 展示 |
|------|------|
| CAPTURED / PROCESSING | 骨架屏 + spinner |
| FAILED | 红色左边框 + 错误简述 + Retry 按钮 |

#### 空状态

📭 Inbox is empty · Capture your first link with the Chrome extension to get started.

### 4.3 Item Detail 页

单列居中布局（max-width 720px）。

```
┌──────────────────────────────────────────────────────┐
│  ← Back to Inbox                                     │
│                                                      │
│  How We Redesigned Our API Gateway                   │
│  engineering.example.com                             │
│  "想了解他们网关的限流和熔断方案"                        │
│  Do first · 85/100 · 2 hours ago                     │
│                                                      │
│  ──── AI Brief ──────────────────────────── Edit ──── │
│  ✦ 文章介绍了限流策略设计——正好回应你对网关方案的关注      │
│  ✦ 提供了熔断机制实现——可直接与当前项目对比适用性          │
│  ✦ 附带新老架构性能数据——为你的技术决策提供客观支撑        │
│  💡 这篇文章的独特价值：用真实案例证明了简单方案的优越性    │
│                                                      │
│  ──── Todos ──────────────────────────── Edit ────── │
│  ☐  阅读限流策略章节              ETA  20m            │
│  ☐  对比现有系统的熔断配置         ETA  30m            │
│  ☐  整理可复用的设计模式笔记       ETA  45m            │
│                                        Total ~1h35m  │
│                                                      │
│  [ Archive ]  [ Open Original ↗ ]  [ 🗑 Delete ]     │
└──────────────────────────────────────────────────────┘
```

**各区块：**

| 区块 | 内容 | 交互 |
|------|------|------|
| Header | Title + domain + intent + priority/score/time | 只读 |
| AI Brief | 3 价值要点（结合 Intent 与文章内容）+ 1 洞察 | 可编辑 |
| Todos | 任务列表 + ETA | 可勾选、编辑、新增/删除 |
| Actions | Archive / Open Original / Delete | Delete 需确认 |

**编辑模式：** 点击 Edit → 文字变为可编辑，右上角切换为 Save / Cancel。

**Todos 交互：**
- 勾选 checkbox → 划线 + 透明度降低
- 全部完成 → 引导 "All done! Archive this item?"

**删除按钮：** 红色，点击弹出 `confirm()` 确认框，确认后删除并返回 Inbox。PROCESSING 状态下禁止删除。

## 5. 状态与反馈

| 场景 | 反馈形式 |
|------|---------|
| Capture 成功 | Popup 画面切换：绿色对勾 + 引导文案 |
| Capture 失败 | Popup 底部红色 toast + Retry |
| 处理中 | 骨架屏 + spinner |
| 处理失败 | 红色左边框 + 错误简述 + Retry |
| 编辑保存 | Toast "Saved ✓"，1.5 秒消失 |
| 归档 | 卡片滑出 + toast "Archived" |
| 恢复 | toast "Restored to Inbox" |
| 删除 | confirm 确认 → 跳转/刷新 + toast "Deleted" |
| 批量操作 | toast 显示操作数量 |

**Toast**：底部居中，距底 24px，深色背景 `#1A1D23`，白色文字，fade in/out。

---

## 6. 响应式策略

| 断点 | 处理 |
|------|------|
| ≥1200px | 完整布局：Sidebar + 宽内容区 |
| 768–1199px | Sidebar 折叠为图标模式（48px），hover 展开 |
| <768px | 暂不适配移动端 |

Extension Popup 固定 360px 宽。

---

## 7. 动效与过渡

| 交互 | 动效 | 时长 |
|------|------|------|
| 卡片悬停 | translateY(-1px) + 阴影加深 | 150ms |
| 页面切换 | 内容区 fade in | 200ms |
| Toast | fade in + translateY(8→0) | 200ms |
| 卡片归档 | slideRight + fadeOut | 250ms |
| 骨架屏 | shimmer 动画 | 1.5s loop |
| 编辑态 | 边框颜色过渡 | 150ms |
