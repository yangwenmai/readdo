# Docs Sync Patch

## 目标
把单点改动扩展为全局一致改动，降低回归。

## 工作流
1. 先定位主改动点（source of truth）。
2. 全局检索同名字段/路径/枚举/状态。
3. 按“必须同步文件”批量修补：contracts、templates、evals、ux/spec。
4. 最后做一次残留检索，确认没有旧值遗留。

## 同步优先级
- 第一优先：契约与 schema。
- 第二优先：模板与 runner 规范。
- 第三优先：PRD/UX/执行计划说明文档。

## 输出要求
- 给出已修改文件清单。
- 明确哪些是 Critical 修复，哪些是一致性补丁。
